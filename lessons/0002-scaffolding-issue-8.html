Scaffolding the monorepo

Completing issue #8 — step by step

What you will build

By the end of this lesson you will have a working monorepo: both packages compile, npm install works from the root, and you are ready to pick up issue #9 (ESLint + Prettier).

Every command runs from the repo root (self-improover/) unless stated otherwise.

Step 1 — Root package.json

Create package.json at the repo root. The only required fields are private: true (prevents accidentally publishing the root) and workspaces: 

{
  "name": "self-improover",
  "private": true,
  "workspaces": [
    "frontend",
    "backend"
  ],
  "scripts": {
    "dev:frontend": "npm run dev --workspace=frontend",
    "dev:backend":  "npm run dev --workspace=backend"
  }
}
Why private: true? Without it, running npm publish by accident would try to publish the root package to the npm registry. private: true makes that an error instead.

Step 2 — Shared TypeScript base config

Create tsconfig.base.json at the root. This is the single source of truth for compiler strictness: 

{
  "compilerOptions": {
    "strict": true,
    "target": "ES2022",
    "esModuleInterop": true,
    "skipLibCheck": true,
    "forceConsistentCasingInFileNames": true
  }
}

skipLibCheck: true skips type-checking inside node_modules/ — this avoids false errors from third-party packages that have imperfect types.

Step 3 — Backend package

Create the backend/ directory and its package.json:

{
  "name": "@self-improover/backend",
  "version": "0.0.1",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "tsx watch src/index.ts"
  }
}

Then create backend/tsconfig.json:

{
  "extends": "../tsconfig.base.json",
  "compilerOptions": {
    "lib": ["ES2022"],
    "module": "NodeNext",
    "moduleResolution": "NodeNext",
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"]
}

And a minimal entry point at backend/src/index.ts:

console.log("backend: ok");
What is tsx? tsx is a lightweight tool that runs TypeScript files directly in Node.js without a separate compile step — like ts-node but faster. You will install it in a moment. It is only used during development; the production build compiles to plain JavaScript.

Step 4 — Frontend package

Create frontend/package.json:

{
  "name": "@self-improover/frontend",
  "version": "0.0.1",
  "private": true,
  "type": "module",
  "scripts": {
    "dev": "vite",
    "build": "tsc && vite build",
    "preview": "vite preview"
  }
}

Then frontend/tsconfig.json:

{
  "extends": "../tsconfig.base.json",
  "compilerOptions": {
    "lib": ["ES2022", "DOM", "DOM.Iterable"],
    "module": "ESNext",
    "moduleResolution": "bundler",
    "jsx": "react-jsx",
    "outDir": "dist",
    "rootDir": "src"
  },
  "include": ["src"]
}

And a minimal entry point at frontend/src/main.tsx:

console.log("frontend: ok");

Step 5 — Install dependencies

Install the minimum dependencies needed to make both packages compile. Run these from the repo root: 

# Backend: TypeScript + tsx runner
npm install --save-dev typescript tsx --workspace=backend

# Frontend: TypeScript + Vite + React
npm install --save-dev typescript vite @vitejs/plugin-react --workspace=frontend
npm install react react-dom --workspace=frontend
npm install --save-dev @types/react @types/react-dom --workspace=frontend

Then install everything with a single root-level install:

npm install

Step 6 — Vite config for the frontend

Vite needs a config file to know you are using React. Create frontend/vite.config.ts: 

import { defineConfig } from "vite";
import react from "@vitejs/plugin-react";

export default defineConfig({
  plugins: [react()],
});

Step 7 — Verify both packages work

Run each package in turn and confirm there are no errors:

# Backend: should print "backend: ok"
npm run dev --workspace=backend

# Frontend: should start the Vite dev server
npm run dev --workspace=frontend

If both run without TypeScript errors, issue #8 is done.
